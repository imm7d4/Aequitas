# US-12.1 - Admin Control Center (Governance & MFA)

**Epic:** Admin & Audit Controls  
**Phase:** Phase 12 - Admin & Audit  
**Status:** Defined (Refined)  
**Priority:** Critical  
**Complexity:** High  

## User Story

As a **Platform Administrator**,  
I want a secure, high-integrity dashboard for platform governance and monitoring,  
so that I can oversee system health, manage high-risk overrides, and enforce **Separation of Duties** with **Step-Up MFA**.

## Acceptance Criteria

### 1. Executive KPIs (Real-Time SLA)
- [ ] **Data Latency**: Metrics must be delivered with a latency of **≤ 5 seconds**.
- [ ] **Streaming Mechanism**: Use WebSockets for volatile metrics (TPM, Concurrent Sessions) and polling (60s) for non-volatile metrics (DAU).

### 2. Metrics Definitions (Deterministic)
- [ ] **TPM (Trades Per Minute)**: Successful executed trades per minute (exclude failed/cancelled).
- [ ] **DAU (Daily Active Users)**: Unique users with ≥ 1 activity in the last 24h (rolling window).
- [ ] **Concurrent Sessions**: Number of active, non-expired JWT sessions.

### 3. Smart Alerts & Configurable Thresholds
- [ ] **Configurable Levels**: Thresholds for error rates, latency, and volume must be editable via Admin Config (INFO/WARNING/CRITICAL).
- [ ] **Standard Deviation Monitoring**: Flag anomalies if deviation > X standard deviations from rolling average.

### 4. Admin Roles & Separation of Duties (SoD)
- [ ] **PlatformAdmin Role**: Manages operational state and approval queues. Cannot modify or verify audit logs.
- [ ] **AuditAdmin Role**: Exclusive access to **Audit Log Forensics**, **Hash Chain Verification**, and **External Anchoring**. Cannot execute financial actions.

### 5. High-Impact Safety & Step-Up MFA
- [ ] **Step-Up Authentication**: Require TOTP/Email MFA for **Global Market Halt**, **Large Approvals**, and **Config Overrides**.
- [ ] **Dual Approval**: Extremely high-risk actions (e.g., Global Halt) require approval from **two independent PlatformAdmins**.
- [ ] **Break-Glass Emergency**: PlatformAdmin can bypass Maker-Checker for a Halt if MFA is provided; triggers immediate **Security Alert** to AuditAdmin.

## Business Analyst Insights
> [!IMPORTANT]
> Real-time latency (≤ 5s) and deterministic KPIs ensure that administrative decisions are based on accurate, non-ambiguous data. Split-Admin architecture prevents any single individual from compromising both the operational state and the forensic trail.
