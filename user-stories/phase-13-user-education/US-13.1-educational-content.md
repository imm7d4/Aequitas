# US-13.1 - Comprehensive Feature Documentation System

**Epic:** User Education & Documentation  
**Phase:** Phase 13 - User Education  
**Status:** Not Started  
**Priority:** High  
**Complexity:** Medium  

## User Story

As a **platform user**,  
I want comprehensive, searchable documentation for every feature in the Aequitas platform,  
so that I can learn how to use features effectively, understand trading concepts, and troubleshoot issues independently.

## Acceptance Criteria

### 1. Documentation Hub
- [ ] Dedicated "Help" or "Docs" section in main navigation
- [ ] Clean, organized layout with sidebar navigation
- [ ] Search functionality across all documentation
- [ ] Breadcrumb navigation for easy orientation
- [ ] Responsive design (works on mobile/tablet)

### 2. Feature Documentation Categories

#### **Getting Started**
- [ ] **Welcome to Aequitas**: Platform overview and key features
- [ ] **Creating Your Account**: Registration process step-by-step
- [ ] **First Login**: Authentication and security setup
- [ ] **Platform Navigation**: Understanding the UI, sidebar, header
- [ ] **Account Setup**: Profile, preferences, and initial configuration

#### **Market Data & Research**
- [ ] **Understanding Instruments**: What are stocks, how they're listed
- [ ] **Market Hours**: Trading sessions, pre-market, post-market
- [ ] **Watchlists**: Creating, managing, and organizing watchlists
- [ ] **Live Price Data**: Real-time quotes, bid/ask, volume
- [ ] **Stock Charts**: Reading candlestick charts, timeframes, zoom/pan
- [ ] **Technical Indicators**: Complete guide to all available indicators
  - Moving Averages (SMA, EMA)
  - RSI (Relative Strength Index)
  - MACD (Moving Average Convergence Divergence)
  - Bollinger Bands
  - Volume indicators

#### **Trading & Orders**
- [ ] **Order Types Explained**:
  - Market Orders: Immediate execution at best price
  - Limit Orders: Execution at specific price or better
  - Stop Orders: Trigger-based orders for risk management
  - Stop-Limit Orders: Combining stop and limit
  - Trailing Stop Orders: Dynamic stop-loss
- [ ] **Placing Your First Order**: Step-by-step guide with screenshots
- [ ] **Order Validity**: DAY, GTC (Good-Till-Cancelled), IOC (Immediate-or-Cancel)
- [ ] **Modifying Orders**: How to change price or quantity
- [ ] **Cancelling Orders**: How to cancel pending orders
- [ ] **Order Status**: Understanding NEW, FILLED, CANCELLED, REJECTED
- [ ] **Order History**: Viewing past orders and trades

#### **Portfolio Management**
- [ ] **Portfolio Overview**: Understanding your dashboard
- [ ] **Holdings**: Viewing positions, quantities, average prices
- [ ] **Profit & Loss (P&L)**: How P&L is calculated
  - Unrealized P&L (open positions)
  - Realized P&L (closed positions)
- [ ] **Position Details**: Viewing transaction history per instrument
- [ ] **Equity Curve**: Tracking portfolio performance over time

#### **Account & Profile**
- [ ] **Profile Settings**: Updating personal information
- [ ] **Security Settings**: Password management, session control
- [ ] **Trading Preferences**: Default order settings, confirmations
- [ ] **Account Finances**: Viewing balance, deposits, withdrawals
- [ ] **Notifications**: Managing notification preferences

#### **Advanced Features**
- [ ] **Short Selling**: How to short stocks, risks, and mechanics
- [ ] **Trade Diagnostics**: Analyzing completed trades
  - MAE (Maximum Adverse Excursion)
  - MFE (Maximum Favorable Excursion)
  - Commission impact
  - Holding duration

### 3. Content Format & Quality

Each documentation article must include:
- [ ] **Clear Title**: Descriptive and searchable
- [ ] **Summary**: 1-2 sentence overview
- [ ] **Step-by-Step Instructions**: Numbered steps with screenshots
- [ ] **Visual Aids**: Screenshots, diagrams, annotated images
- [ ] **Examples**: Real-world scenarios and use cases
- [ ] **Tips & Best Practices**: Pro tips highlighted in callout boxes
- [ ] **Common Mistakes**: What to avoid
- [ ] **Related Articles**: Links to related documentation
- [ ] **Last Updated**: Timestamp for freshness

### 4. Search & Navigation
- [ ] Full-text search across all articles
- [ ] Search suggestions as you type
- [ ] Filter by category
- [ ] "Popular Articles" section
- [ ] "Recently Updated" section
- [ ] Table of contents for long articles

### 5. Trading Glossary
- [ ] Comprehensive A-Z glossary of trading terms
- [ ] Each term includes:
  - Definition
  - Example usage
  - Related terms
  - Link to detailed article (if applicable)
- [ ] Search and filter functionality
- [ ] Alphabetical index

### 6. FAQ Section
- [ ] Frequently Asked Questions organized by category
- [ ] Quick answers with links to full documentation
- [ ] "Was this helpful?" feedback buttons
- [ ] Contact support link for unanswered questions

## Technical Requirements

### Backend

#### New Models
```go
type Documentation struct {
    ID          uint      `gorm:"primaryKey" json:"id"`
    Slug        string    `gorm:"uniqueIndex;not null" json:"slug"`
    Title       string    `gorm:"not null" json:"title"`
    Category    string    `gorm:"index" json:"category"`
    Summary     string    `json:"summary"`
    Content     string    `gorm:"type:text" json:"content"` // Markdown
    Order       int       `json:"order"` // Display order within category
    Tags        string    `json:"tags"` // Comma-separated for search
    ViewCount   int       `gorm:"default:0" json:"view_count"`
    IsPublished bool      `gorm:"default:true" json:"is_published"`
    UpdatedAt   time.Time `json:"updated_at"`
    CreatedAt   time.Time `json:"created_at"`
}

type GlossaryTerm struct {
    ID           uint   `gorm:"primaryKey" json:"id"`
    Term         string `gorm:"uniqueIndex;not null" json:"term"`
    Definition   string `gorm:"type:text" json:"definition"`
    Example      string `json:"example,omitempty"`
    RelatedTerms string `json:"related_terms"` // Comma-separated
    Category     string `json:"category"`
}

type FAQ struct {
    ID       uint   `gorm:"primaryKey" json:"id"`
    Question string `gorm:"not null" json:"question"`
    Answer   string `gorm:"type:text" json:"answer"`
    Category string `gorm:"index" json:"category"`
    Order    int    `json:"order"`
    Helpful  int    `gorm:"default:0" json:"helpful_count"`
}
```

#### New Service: `DocumentationService`
- **Method**: `GetAllDocs(category string) ([]Documentation, error)`
  - Fetch published documentation by category
  
- **Method**: `GetDocBySlug(slug string) (*Documentation, error)`
  - Fetch specific article
  - Increment view count
  
- **Method**: `SearchDocs(query string) ([]Documentation, error)`
  - Full-text search on title, summary, content, tags
  
- **Method**: `GetGlossary() ([]GlossaryTerm, error)`
  - Fetch all glossary terms
  
- **Method**: `GetFAQs(category string) ([]FAQ, error)`
  - Fetch FAQs by category

#### New Endpoints
- `GET /api/docs` - List all documentation
- `GET /api/docs/category/:category` - Docs by category
- `GET /api/docs/:slug` - Get specific article
- `GET /api/docs/search?q=query` - Search documentation
- `GET /api/glossary` - Get all glossary terms
- `GET /api/faqs` - Get all FAQs
